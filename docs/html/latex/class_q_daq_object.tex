\doxysubsection{Detailed Description}
Base class of all \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} objects.

\mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} is derived from Q\+Object and thus inherits all the Qt machinery, i.\+e., properties, meta-\/objects, signals \& slots, etc.

\mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} objects are organized in a tree-\/like structure like Q\+Objects. However, the children of a \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} are ordered ,whereas those of a Q\+Object are not. The ordering is necessary for \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} because \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} objects take part in loops where the order of execution is essential.

To create the \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} object tree structure, the \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} class offers an interface similar to the \mbox{\hyperlink{class_node}{Node}} interface of Document Object Model (D\+OM) Level 1. The interface is defined by this \mbox{\hyperlink{class_q_daq_object_domapi}{group of functions}}.

Initially a \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} is created without a parent and only its name needs to be specified. The object can than be incorporated into the \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} hierarchy by one of the \mbox{\hyperlink{class_q_daq_object_domapi}{D\+OM A\+PI functions}}.

There is a static single root object at the top of the tree structure. The root object is accesible by \mbox{\hyperlink{class_q_daq_object_a9f3d71496598a421c34fc099bcfcb1ae}{Q\+Daq\+Object\+::root()}} and it of type \mbox{\hyperlink{class_q_daq_root}{Q\+Daq\+Root}}.

Descendants of \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} can be serialized to \href{https://support.hdfgroup.org/HDF5/}{\texttt{ H\+D\+F5 files.}} A \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} object with its children can be written/read by means of h5write() and h5read(), respectively.

H\+D\+F5 serialization is done according to the following rules\+:
\begin{DoxyItemize}
\item Each object is written in the H\+D\+F5 file as a \mbox{\hyperlink{namespace_h5}{H5}} group. The name of the object becomes the name of the group.
\item Properties and data of the object are written as datasets of the group. \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} descendants reimplement \mbox{\hyperlink{class_q_daq_object_a0ae6d445c5b86bfae21ee5bf743f0d82}{writeh5()}}/readh5() where necessary.
\item Child objects are written as sub-\/groups 
\end{DoxyItemize}
