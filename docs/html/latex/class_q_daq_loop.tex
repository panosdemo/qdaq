\doxysubsection{Detailed Description}
A class encapsulating a software loop. 

\mbox{\hyperlink{class_q_daq_loop}{Q\+Daq\+Loop}} is used to execute a number of child-\/jobs and/or child-\/loops.

By setting the class properties we can arrange for a number of different execution scenarios\+: repetitive with given period repeat for ever or for a given number of cycles skip execution every delay cycles single shot

A \mbox{\hyperlink{class_q_daq_loop}{Q\+Daq\+Loop}} that has no \mbox{\hyperlink{class_q_daq_loop}{Q\+Daq\+Loop}} ancestor is considered a \char`\"{}top level loop\char`\"{} (\mbox{\hyperlink{class_q_daq_loop_a2b1763fd29487f3e72d943598b8205aa}{is\+Top()}} returns true). Otherwise the loop is a child-\/loop.

When \mbox{\hyperlink{class_q_daq_loop_a87548dcf7b27cb4affbabd5908e0d17f}{arm()}} is called on a top level loop, a new \mbox{\hyperlink{class_q_timer_thread}{Q\+Timer\+Thread}} is spawned that calls \mbox{\hyperlink{class_q_daq_loop_aae1fa092929e526545300c29a0501649}{exec()}} at each timer repetition.

If \mbox{\hyperlink{class_q_daq_loop_a87548dcf7b27cb4affbabd5908e0d17f}{arm()}} is called on a child loop, then it simply arms all child jobs. The \mbox{\hyperlink{class_q_daq_loop_aae1fa092929e526545300c29a0501649}{exec()}} function of a child loop is called from the top level loop thread.

To use \mbox{\hyperlink{class_q_daq_loop}{Q\+Daq\+Loop}}, create the object and build underneath it a tree of child-\/jobs and child-\/loops that perform the required tasks in the correct order. Then attach the loop to the \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} tree either as a child of the root object (\char`\"{}qdaq\char`\"{}) or under another object that does not have a \mbox{\hyperlink{class_q_daq_loop}{Q\+Daq\+Loop}} anchestor. Thus the loop becomes top level.

{\footnotesize\ttfamily bool Q\+Daq\+Loop\+::exec (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}



Called when a loop is executed. 

In a top level loop this function is called at each repetition of the timer thread.

In a child-\/loop \mbox{\hyperlink{class_q_daq_loop_aae1fa092929e526545300c29a0501649}{exec()}} is called from the parent loop according to the order of the child-\/loop in the tree structure.

This function first locks the mutexes of all child jobs to prevent access from other threads. Then it calls \mbox{\hyperlink{class_q_daq_job_a39c072085e85df73a1608e9d347fb518}{Q\+Daq\+Job\+::exec()}} which runs all child jobs. Finally it unlocks the mutexes in the reverse order as they were locked.

The signals \mbox{\hyperlink{class_q_daq_object_aededf89ebbc4b02e9d311cd73b6d13aa}{update\+Widgets()}} and \mbox{\hyperlink{class_q_daq_object_a9f7bca76048cd5f6fe6b1078da3e3b33}{properties\+Changed()}} are emitted at each valid repetition.
