\doxysubsection{Detailed Description}
A class representing a D\+AQ device. 

It is used in combination with a \mbox{\hyperlink{class_q_daq_interface}{Q\+Daq\+Interface}} in order to communicate with the device.

In order to start communications with a device, first instantiate the interface and device objects, asign the interface to the device and then open() the interface and bring the device online with \mbox{\hyperlink{class_q_daq_device_a6c23ad1696d0b2c13ed316f093d7a856}{on()}}.

Performs internal initialization for the job.

It is called by the \mbox{\hyperlink{class_q_daq_job_a3af371946f6916812fd6ad03904b2293}{set\+Armed()}} function.

This function can be reimplemented to perform specific initialization for a job. The parent class \mbox{\hyperlink{class_q_daq_device_ac93ade30efbd57199449afae4b08ac8e}{arm\+\_\+()}} function should be called afterwards.

In the default implementation, if a \mbox{\hyperlink{class_q_daq_job_aba27ebc276314277e303cc8c97e9f439}{loop\+Engine()}} exists the script code is checked against the script engine for errors.

If initialization is sucessfull it returns true, otherwise the function returns false.

Detach this \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} from the \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} tree.

This function is called just before the object becomes detached from the \mbox{\hyperlink{namespace_q_daq}{Q\+Daq}} tree.

In the base class implementation it first calls \mbox{\hyperlink{class_q_daq_device_a85104e6fbae6f9d77afc7777d171943c}{detach()}} for each child \mbox{\hyperlink{class_q_daq_object}{Q\+Daq\+Object}} and then signal root that the object is removed from the qdaq tree.

Subclasses may reimplement this function to perform needed actions before detaching.
